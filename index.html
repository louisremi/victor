Limitations

Raphael supports only a subset of SVG features (for the sake of VML compatibility). Here's a list of the most notable limitations:

  • no <g> elements (see Paper.set() alternative),
  • no <switch> elements,
  • no filter elements (blur, shadow, etc.),
  • limited support of stroke-dasharray (see Element.attr()),
  • no support of marker-* (see arrow-end in Element.attr()),
  • specific syntax for gradients, limited to simple gradients for <path> elements (see Gradients in Element.attr()).

In addition, Victor doesn't yet fully support the following Raphael features:

  • conversion of <g> elements (planned),
  • conversion of transforms (planned),
  • approximative conversion of gradients (planned),
  • approximative conversion of marker-end (planned),
  • clip-rect, stroke-dasharray, stroke-linejoin, stroke-linecap, stroke-miterlimit (trivial).

Preparing svg files

Some tweaks can be applied to svg files to improve compatibility with the web. Here's what Inkscape (free-OSS editor) allows:

  • ungroup all groups of elements,
  • apply transforms to paths by selecting them all, grouping them and ungrouping them,
  • convert other transformed elements (rectangles, ellipses, circles, text) to paths,
  • convert text elements winth non-standard font-family to paths,
  • replace gradients with plain colors,
  • remove filters.

A screencast should be posted to help unexperienced users.
